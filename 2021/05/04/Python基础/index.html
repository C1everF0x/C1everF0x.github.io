Python基础

写在前面

随便学学，反正选了这个课，就当巩固基础了，而且更早之前有过一版 Python 的学习笔记了，当时是跟着菜鸟教程学的，但是这个课里面有没学到的，索性再整理一版笔记

写得会比较精简，太太基础的就不写了

反正想知道什么东西，直接help(东西)加上百度就行

实际写脚本的过程中，感觉最头疼的不是去学习使用一个新的库，而是在debug 过程中遇到最
多的是输入参数和输出参数的数据类型问题，需要输入什么数据类型，返回出来的是什么数据
类型，很头疼

ch2 变量和基本数据类型

  • type()：输入变量，返回变量的数据类型

  • int(x, d)：输入字符串 x ，转换为对应 d 进制的数为十进制的数，返回 int 类型

  • bin(x)：输入 int 型十进制参数 x 转换为二进制 str 类型结果返回

  • oct(x)：输入 int 型十进制参数 x 转换为八进制 str 类型结果返回

  • hex(x)：输入 int 型十进制参数 x 转换为十六进制 str 类型结果返回

  • 不需要反斜杠转义用原生字符串r'xxxxxx'或用两个反斜杠，这个在输入文件路径时会用到

  • 字符串可以用索引访问，没有单个字符的说法

  • 字符串拼接用 +

  • 内置置逆函数reversed()，使用函数后返回的是可迭代类型

  • 切分与连接，''.join()与str.split(x)，split 只能切割字符串类型，.join 可以接受元组列表字符串range等可迭代类型

切分、连接、置逆在字符串、元组、列表等数据类型中比较常用，基本都是返回一个可迭代类型

ch3 复合数据结构

  • 序列：指包含多项数据的数据结构，数据按顺序存储，字符串、列表、元组、字典等都是序列

  • 不可变对象：数字、字符串、元组

  • 可变对象：列表、集合、字典

    • 本质区别：

      当给不可变对象重新传值的时候，不是修改了变量的值，而是重新生成了一个新的对象，然后再让变量指向新的内存地址，其内存地址发生了改变

      当给可变对象重新传值的时候，是真正修改了对象的值，没有重新生成一个新的对象，变量前后指向的是同一个内存地址，其内存地址没有发生改变

列表和元组常用方法

  • 查找和计数，index(x)和count(x)方法，查找返回 x 第一次出现位置，计数返回 x 出现的次数，没有返回0

  • 序列封包：把多个值赋给一个变量，Python 将这些值封装成元组或列表

  • 序列解包：将序列直接赋值给多个变量，其中的元素依次赋值给每个变量

  • 列表常用方法

    append() 列表尾部添加元素，可以是元组或列表
    insert() 指定位置插入
    extend() 将一个列表追加在列表尾部
    del() 删除
    remove(x) 根据元素 x 的值本身删除
    clear() 清空列表
    reverse() 列表置逆
    sort() 列表排序，默认从小到大，添加参数 reverse=True 后为从大到小
    pop() 出栈

  • 深拷贝和浅拷贝

    • 直接赋值，浅拷贝，前后两个列表引用的是同一段内存里面的 list 对象，如果其中一个列表发生改变，另一个也会发生改变

    • 使用 copy 方法，浅拷贝，如果列表中嵌套了可变序列，则拷贝前后两个列表共享这个可变序列，但是其他成员改变不会两个同时改变

    • 深拷贝使用 deepcopy 方法，就是完完全全的两个对象，拷贝之后互不相干，各自拥有自己的内存

  • 元组常用方法

    元组很常用的就是和列表之间相互转换，list()和tuple()，因为元组自带的方法很少，index()和count()

字典常用方法

  • 常用于具有唯一性的数据，使用字典的数据结构存储方便进行快速查找

  • get(x)方法返回指定键 x 对应的值，键不存在返回 None

  • items()方法返回字典的键值对，用元组分装

  • keys()方法返回字典的键

  • values()方法返回字典的值

ch4 流程控制

  • 列表生成式，[表达式 for 循环计数器 in 可迭代对象]，表达式是要生成的元素，循环计数器记录循环次数的变量

    要查看文件夹内所有的绝对路径

  • 生成器，就是不一次性生成完的列表生成式，更加省内存

  • 迭代器，能够用于 for 循环的对象都叫可迭代对象，基本六个数据类型里面除了 int 数字类型，其他都是可迭代对象，还有就是 generator 类对象，包括生成器和带 yield 的 generator function

    判断是否为可迭代对象用 collections 模块里面的 Iterable 类型和isinstance()函数来判断
