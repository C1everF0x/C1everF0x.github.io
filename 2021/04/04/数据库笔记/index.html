第二章

基础概念

  • 关系：是一种数据结构，一般抽象成一张二维表

  • 域：一组具有相同数据类型的值的集合，用人话说，就是表中属性的范围，比如年龄范围（0-99）
  • 元组：二维表的一行就是一个元组
  • 属性：一列为一个属性，列名为属性名

关系操作

  • 选择、投影、并、差、笛卡尔积为五种基本操作
  • 结构化查询语言 —— SQL，集查询、数据定义语言、数据操纵语言和数据控制语言于一体的关系数据语言

关系完整性

  • 实体完整性、参照完整性、用户定义完整性三类完整性，其中 实体完整性和参照完整性 是必不可少的，称为 关系的两个不变性

  • 实体完整性：若属性 A 是基本关系 R 的主属性，则 A 不能为空（是所有的主属性）

  • 参照完整性：设 F 是基本关系 R 的一个或一组属性，但不是 R 的码，Ks 是基本关系 S 的主码。如果 F 与 Ks 相对应，则称 F 是 R 的外码，并称基本关系 R 为参照关系，基本关系 S 为被参照关系或目标关系，其中 关系 R 和 S 不一定是不同的关系。

    用通俗的语言解释，不同或相同的关系中，存在属性的交叉引用，一个关系中的属性的取值，需要去参考另外一个或者自身的另外一个属性的取值，就是存在参照关系，需要遵守参照完整性规则

    参照完整性规则即：若属性 F 被关系 R 中元组引用，F 与关系 S 的主码 Ks 相对应，则 R 中每个元组中 F 的值只有两种可能：1、空值，2、等于 关系 S 中某个元组的主码值

  • 用户自定义完整性：针对某一具体的关系数据库的约束条件

模式

  • 模式指的是关系型数据库的三级模式结构，分为外模式、模式和内模式，其中模式在中间，一个数据库中可以建立多个模式，一个模式下通常包括多个表、视图、索引等数据库对象，我理解的模式就是相当于一个用户，这个用户管理了多个表

关系型数据库

  • 建立在关系模型基础上的数据库，借助于集合代数等数学概念和方法来处理数据库中的数据
  • 主流数据库：Oracle、Microsoft SQL Server、MySQL等

专业术语

  • 数据库：一些关联表的集合
  • 数据表：表是数据的矩阵，像一个表格
  • 列(col)：具有相同数据类型的数据的集合，一列包含了相同类型的数据
  • 行(row)：每一行用来描述某条记录的具体信息，一行是一组相关的数据
  • 值(value)：行的具体信息, 每个值必须与该列的数据类型相同
  • 键(key): 键的值在当前列中具有唯一性。
  • 冗余：存储两倍数据，冗余降低了性能，但提高了数据的安全性。
  • 主键：主键是唯一的。一个数据表中只能包含一个主键。你可以使用主键来查询数据。
  • **外键：**外键用于关联两个表。
  • 复合键：复合键（组合键）将多个列作为一个索引键，一般用于复合索引。
  • **索引：**使用索引可快速访问数据库表中的特定信息。索引是对数据库表中一列或多列的值进行排序的一种结构。类似于书籍的目录。
  • 参照完整性: 参照的完整性要求关系中不允许引用不存在的实体。与实体完整性是关系模型必须满足的完整性约束条件，目的是保证数据的一致性。

转自菜鸟教程：https://www.runoob.com/mysql/mysql-tutorial.html

SQL语言

  • 介于关系代数和关系演算之间的结构化查询语言，关系型数据库基本上都用这个语言

Mysql

  • 非常流行的关系型数据库管理系统

常见操作命令

  • mysql -V (V要大写)：查询mysql版本

  • 开启、关闭MySQL服务用phpstudy

  • mysql -h 主机名 -u 用户名 -p

  • -h : 指定客户端所要登录的 MySQL 主机名, 登录本机(localhost 或 127.0.0.1)该参数可以省略;
    -u : 登录的用户名;
    -p : 告诉服务器将会使用一个密码来登录, 如果所要登录的用户名密码为空, 可以忽略此选项。

  • 出现mysql>加光标闪烁表示登陆成功

  • 登陆时通过传递参数 mysql -u root -p root --prompt 命令提示符名称

    • \h 服务器名称
    • \D 完整的日期
    • \d 当前数据库
    • \u 当前用户

  • 添加MySQL用户，在mysql数据库中的user表中添加新用户

注意：在 MySQL5.7 中 user 表的 password 已换成了authentication_string。

  		password() 加密函数已经在 8.0.11 中移除了，可以使用 MD5() 函数代替。

管理MySQL命令

  • use 数据库名：选择要操作的数据库
  • show databases：列出数据库列表
  • show tables：显示指定数据库的所有表，使用该命令前需要使用 use 命令来选择要操作的数据库。

  • show columns from 数据表：显示数据表的属性，属性类型，主键信息 ，是否为 NULL，默认值等其他信息。

    也可用desc 数据表名字;

  • show index from 数据表：显示数据表的详细索引信息，包括PRIMARY KEY（主键）。
  • show table status like [FROM db_name] [LIKE ‘pattern’] \G：该命令将输出Mysql数据库管理系统的性能及统计信息。

MySQL删除数据

  • 如果没有指定where语句，表中所有内容都会被删除
  • 可以在where语句中设置任何条件
  • 可以在一个单独表中一次性删除记录

delete，drop，truncate 都有删除表的作用，区别在于：

  • delete 和 truncate 仅仅删除表数据，drop 连表数据和表结构一起删除，打个比方，delete 是单杀，truncate 是团灭，drop 是把电脑摔了。
  • delete 是 DML 语句，操作完以后如果没有不想提交事务还可以回滚，truncate 和 drop 是 DDL 语句，作完马上生效，不能回滚，打个比方，delete 是发微信说分手，后悔还可以撤回，truncate 和 drop 是直接扇耳光说滚，不能反悔。
  • 执行的速度上，drop>truncate>delete，打个比方，drop 是神舟火箭，truncate 是和谐号动车，delete 是自行车。

总结一下就应该是：

删除表内数据，用delete，带条件时不会释放磁盘空间，下次插入数据任然可以用这部分空间；不带条件全部删除时，MyISAM会立刻释放磁盘空间，InnoDB不会释放，delete操作以后，用optimize table 表名会立刻释放磁盘空间

清除表内数据，保存表的结构，用truncate，会立即释放磁盘空间

删除表用drop，会立刻释放磁盘空间

1、当你不再需要该表时，用 drop

2、当你仍要保留该表，但要删除所有记录时，用 truncate

3、当你要删除部分记录时，用 delete

MySQL好用的like子句

where语句查询使用=来精确查找，like子句可以达到范围查询/模糊匹配的效果

  • 在where子句中使用like子句

  • like可以代替=

  • %代表任意字符

    _表示一个字符

    []：要求匹配内容为括号内出现的字符、字符串或范围的任意一个

    [^]：要求匹配内容为不在括号内出现的字符、字符串或范围的任意一个

  • 可用and或or指定一个或多个条件

  • delete、update命令中也可用where…like语句

‘%C1everF0x’ #以C1everF0x结尾的数据
‘C1everF0x%’ #以C1everF0x开头的数据
‘%C1everF0x%’ #含有C1everF0x的数据
‘_C1everF0x_’ #三位且中间字母是C1everF0x的
'_C1everF0x #两位且结尾字母是C1everF0x的
‘C1everF0x_’ #两位且开头字母是C1everF0x的

select * from position where name like ‘C1everF0x%’ # 查询以C1everF0x字段开头的信息

select * from position where name like ‘%C1everF0x%’ # 查询包含C1everF0x字段的信息

select * from position where name like ‘%C1everF0x’ # 查询以C1everF0x结尾的信息

MySQL union操作（联合查询）

  • union：把不同表中相同列中查询的数据展示（不包括重复的数据）
  • union all：把不同表中相同列中查询的数据展示（包括重复的数据）

MySQL连接操作

  • inner join：获取两表中字段匹配关系记录，相当于a表∩b表
  • left join（左连）：获取左表所有记录，相当于a表∪（a表∩b表）
  • right join（右连）：获取右表所有记录，相当于b表∪（a表∩b表）
