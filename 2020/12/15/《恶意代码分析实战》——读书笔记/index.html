《恶意代码分析实战》——学习笔记

发表于2020-12-15|更新于2021-01-09|学习笔记
|阅读量:

写在前面

老师分享了一本很好的书，整点笔记留下来，以后要是忘了里面的知识可以及时翻阅

0x00 恶意代码分析技术入门

啥是恶意代码

恶意代码：无论以什么方式（病毒、木马、蠕虫、内核套件、勒索软件、间谍软件等）对用户、计算机或网络造成破坏的软件，都叫恶意代码

大概分个类：

后门：攻击者留在受害主机上方便进行远控的连接方法

僵尸网络：一片被同一个攻击者操控的受害主机形成的网络

下载器：用来下载其它恶意代码的恶意代码，本身可能无害，静态分析时难以发现

间谍软件：从受害主机上收集信息，如键盘记录器，嗅探器等

启动器：用来启动其他恶意程序的恶意代码

内核套件：用来隐藏其他恶意代码的恶意代码

勒索软件：顾名思义

蠕虫\计算机病毒：可以自我复制和通过网络传播感染其他计算机的恶意代码

分析恶意代码的技术

目的

分析清楚软件到底干了什么，定位出所有受感染的主机和文件

特征码识别

反病毒软件特征码：通常是识别恶意代码本身的特性，比如使用危险的系统调用函数

基于主机的特征码：也称感染迹象，检测恶意代码生成或修改的文件，或者对注册表的修改、删除等操作，主要关注恶意代码对系统做了什么

网络特征码：监测网络流量来检测恶意代码

静态分析

  • 在不运行恶意代码的情况下进行分析

通过逆向分析恶意代码，通常把可执行文件反汇编出机器指令，分析恶意代码让CPU做了什么

动态分析

  • 在运行恶意代码的情况下进行分析

通过使用调试器进行动态调试来检查恶意软件运行时的内部状态

分析通用规则

  1. 不要过于陷入细节
  2. 学会用不同的工具解决问题
  3. 耐心、细心、坚持

0x01 静态分析基础技术

拿不同的杀毒软件扫

  • 杀毒软件主要依靠自己多年来积累的病毒文件特征库，以及基于行为与模式匹配的分析的方式来识别可疑文件
  • 缺点是恶意代码作者能够针对性的隐藏自己的特征，用一些奇技淫巧来绕过杀毒软件对特征码的检测，行话称之为免杀

哈希值——恶意代码的指纹

  • 恶意代码的样本通过一个哈希程序，能够产生一段唯一标识这个样本的哈希值，类似于人类的指纹
  • MD5、SHA-1算法是常用的哈希函数
    • 恶意代码指纹能够共享给其他逆向师傅，在线搜索这个指纹看看其他师傅是不是已经发现了

查找字符串

  • strings程序可以搜索一个程序中的可打印字符串（一般以Unicode格式存储），所谓程序中的字符串，就是指一些打印出来的消息、链接的URL/IP地址（可能是病毒连外网的地址）、复制文件到某个特定位置、还有一些调用的库函数等等，这些可见的字符串都会成为线索，来分析病毒的行为特征

微软使用“双字节字符串”来描述 Windows 中的 Unicode 字符串

ASCII 每个字符使用 1 个字节， Unicode 每个字符使用 2 个字节，结尾都以 NULL结束符 表示字符串的完结

ASCII存储形式

B A D NULL Terminator
42 41 44 00

Unicode存储形式

B A D NULL Terminator
42 00 41 00 44 00 00 00

加壳与混淆

加壳

  • 加壳程序属于混淆程序的一种，加壳后的程序会被压缩

  • 加壳程序运行的时候，会首先运行一小段脱壳代码来解压缩加壳的文件，然后再运行原文件，问题就在于加壳后的可执行程序，拿strings程序来跑，查到的可见字符串很少很少，加大了静态分析的难度

  • 可用PEiD 检测加了什么壳或者程序用什么语言编译的，UPX脱壳程序也很流行

混淆

  • 混淆程序是指恶意代码作者尝试隐藏其执行过程的代码

可移植执行（PE）文件格式

  • Windows可执行文件、对象代码、DLL文件使用的标准格式，本质是一种数据结构
  • PE文件是一个以文件头开始，其中包括代码信息、应用程序类型、所需的库函数与空间要求

链接库与函数

动态链接和静态链接的区别我的理解就是，动态链接在内存中只存在某个模块的唯一一份代码，通过虚拟存储器的内存映射机制将物理地址空间中的代码映射到不同进程的虚拟地址空间中，链接后把库删了就跑不了了，静态链接是在磁盘中和内存中都有模块的两份副本，链接后删掉了库，还能跑

  • Windows不常用，UNIX、Linux常用，库被链接到可执行程序时，全部代码会被复制到可执行文件中
  • Windows下：.lib 后缀为静态库，Linux下：.a后缀为静态库

运行时链接

  • 恶意代码中常用，只有当需要使用函数时，才链接到库

动态链接

  • 程序启动时，就会链接
  • Windows下：.dll 后缀为动态库（动态链接库（Dynamic Linking Library）），Linux下：.so后缀为动态库

编译的过程

编译分为 3 步，首先对源文件进行预处理，这个过程主要是处理一些 # 号定义的命令或语句（如宏、 #include 、预编译指令 #ifdef 等），生成 *.i 文件；然后进行编译，这个过程主要是进行词法分析、语法分析和语义分析等，生成 *.s 的汇编文件；最后进行汇编，这个过程比较简单，就是将对应的汇编指令翻译成机器指令，生成可重定位的二进制目标文件，这就是编译的过程

Dependency Walker软件

  • 用来列出可执行文件的动态链接函数

常见的DLL

DLL 描述
Kernel32.dll 核心系统功能，如访问和操作内存、文件和硬件
Advapi32.dll 对核心 Windows 组件的访问，如服务管理器和注册表
User32.dll 包含所有用户界面组件，如按钮、滚动条等
Gdi32.dll 包含图形显示和操作的函数
Ntdll.dll Windows 内核接口，一般通过 kernel32.dll 间接导入
Wsock32.dll和Ws2_32.dll 联网的DLL，访问这俩DLL一般就是联网了
Wininet.dll 更高层的网络函数，如FTP、HTTP、NTP等

Windows函数命名约定

Ex结尾：如 CreateDirectoryEx，微软更新一个函数，而且新老函数不兼容的时候，会保留老函数，给新函数结尾加一个Ex标识是新函数，更新两次则有两个Ex

A或W结尾：以 A 结尾的输入参数类型为 ASCII 字符串，以 W 结尾的输入参数类型为宽字符字符串

导入函数

  • PE文件头中包含了可执行文件使用的特定函数相关信息

导出函数

  • DLL和EXE的导出函数，用来与其他程序或代码进行交互
