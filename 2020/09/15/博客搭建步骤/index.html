博客搭建踩坑经历
发表于2020-09-15|更新于2020-09-17|生活杂记
|阅读量:

搭这个博客真的是踩坑一堆，现在搞好了以后才发现这么简单，就几个命令行的事情

安装必备软件

Nodejs

Nodejs 下载地址:这里是官网，点击直达

目前的Nodejs版本里面已经是包含了npm，所以我们不需要额外的安装了

Git

Git 下载地址:这里也是官网，点击直达

安装的时候除了看清楚安装到哪个盘什么路径，无脑下一步就完事了

安装完以后打开命令行看看版本

这样就成功了

hexo

找一个盘，新建一个文件夹，改好名字，比如博客名's blog，这个地方就是以后博客存放的

  1. 进入这个博客文件夹，鼠标右键
  2. 点击git bash here
  3. 命令行中输入npm install -g hexo-cli（全局安装hexo）
  4. 等待
  5. 依旧是同一个命令行，输入hexo init（文件夹初始化，部署hexo）

到这里已经把博客部署在本地了

博客文件夹的目录大概就是这样，因为是后面才想起来写这篇博客，已经忘记当时最初是什么样的了

配置博客

几个小细节

  • git bash here都进入博客文件夹以后右键进入这个命令行
  • hexo的相关的命令都要在上一条的命令行里面进行
  • .yml文件的格式非常严格，一定要注意的就是空格，冒号后面一定要接空格，冒号也一定要用英文冒号，不然一定报错
  • 每一次写完新的博客，或者配置完博客、主题，都要hexo g然后再hexo d
  • 博客主题、博客文章里面的图片路径填本地的没用，需要去找一个图床网站，把图片上传上去，然后用返回的图片url当地址

基础配置

配置的文件在博客文件夹里面的_config.yml

hexo 基础命令

命令 意义
hexo init 初始化博客
hexo server 本地运行博客
hexo new “文章名字” 新建一篇文章
hexo clean 清理缓存
hexo g 生成静态文件
hexo d 部署到云端

把博客部署到GitHub上

  1. 创建GitHub账号

  2. 创建一个仓库，仓库名字必须是：<Github账号名称>.github.io

    这里就意味着，GitHub账号名字就一定得是博客的名字

  3. 本地的hexo和GitHub连接

    • 安装hexo-deployer-git插件，即在博客文件夹内git bash here后输入npm install hexo-deployer-git --save

    • 添加SSH key

      • 创建一个SSH key，即在博客文件夹内git bashhere后输入ssh-keygen -t rsa -C "邮箱地址"，回车三下即可

      • 添加SSH密钥到GitHub，先去C:\Users\Administrator\.ssh\id_rsa.pub，用记事本打开这个文件，复制内容，进入GitHub，在title里面备注Blog表示是博客的SSH，key里面粘贴刚刚复制的内容

      • 测试是否添加成功，命令行里面输入ssh -T git@gtihub.com，看到You've successfully xxxxx表示成功

    • 进入博客文件夹打开_comfig.yml，在文件末尾修改

      deploy:
      type: git
      repo: git@github.com:<Github账号名称>/<Github账号名称>.github.io.git
      branch: master

  4. 在老地方依次执行hexo clean——>hexo g——> hexo d

  5. 打开网址输入xxxxxx.github.io即可看到博客
