老是记不住的东西

发表于2020-09-13|更新于2020-11-17
|阅读量:

有些细碎的知识点老是记不住，（难道是老年痴呆提前了？）经常学着学着就忘了，然后又要花时间去搜一搜，现在整一篇博客专门记录，以后随时更新

位（bit）

  • 一个二进制数据 0 或 1 ，是1bit

字节（byte）

  • 储存空间的基本计量单位

  • 1 byte = 8 bit

  • 一个英文字符占一个字节

    即 1 字母 = 1 byte = 8bit

  • 一个汉字占两个字节

    即 1 汉字 = 2 byte = 16bit

  • 中文输入法时标点符号占两个字节

    即 1 中文标点 = 2 byte = 16bit

  • 英文输入法时标点符号占一个字节

    即 1 英文标点 = 1 byte = 8bit

  • mysql 4.0版本以下，varchar(50)就代表 50 个字节，如果存放utf-8汉字时，只能存放 16 个（每个汉字 3 字节）

  • mysql 5.0版本以上，varchar(50)指的是 50 字符，无论存放的是数字、字母还是uft-8汉字（每个汉字 3 字节），都可以存放 50 个

  • 8 位的第一位是符号位
    即：0000 0001 表示数字 1 ，1000 0000 表示 -1 ，正数最大位 0111 1111 表示数字 127 ，负数最大 1111 1111 表示数字 -128
    这是二进制原码的情况

  • 一个数为正，则它的原码、反码、补码相同
    一个数为负，则符号位为 1 ，其余各位是对原码取反，然后整个数加 1

字符（不是标准的计量单位）

  • 各种文字和符号的总称，包括各国家文字、标点符号、图形符号、数字等
  • utf-8 编码下，一个汉字占3个字节
  • gbk编码下，一个汉字占2个字节

Python浮点型字符串转整型字符串

  • 没办法直接用int()转换，需要先转成float型再转成int型

位运算

  • a = 60 二进制为 0011 1100

    b = 14 二进制为 0000 1110

符号 解释 例子
& 按位与：同 1 为 1，不同则 0 a & b = 12，二进制解释：0000 1100
| 按位或：有 1 为 1 ，同 0 则 0 a | b = 62，二进制解释：0011 1110
^ 按位异或：不同为 1 ，相同为 0 a ^ b = 50，二进制解释：0011 0010
~ 按位取反： 1 变成 0 ，0 变成 1 ，其中~a = -a - 1 ~b = -15，二进制解释：1111 0001 (这是补码形式)
<< 左移动：各二进制位全部左移若干位，高位丢弃，低位补 0 a << 2 = 240，二进制解释：1111 0000
>> 右移动：各二进制位全部右移若干位，高位丢弃，低位补 0 a >> 2 = 15，二进制解释：0000 1111

python的pydoc查看类和方法结构

  • pyton3 -m pydoc -p 1234

    通过访问 http://localhost:1234 来访问查看文档

python中__name__是什么

name 是属于 python 中的内置类属性，就是它会天生就存在于一个 python 程序中，代表对应程序名称。

比如所示的一段代码里面（这个脚本命名为 pcRequests.py），我只设了一个函数，但是并没有地方运行它，所以当 run 了这一段代码之后我们有会发现这个函数并没有被调用。但是当我们在运行这个代码时这个代码的 name 的值为 main （一段程序作为主线运行程序时其内置名称就是 __main__)

看到这里应该能明白，自己的 name 在自己用时就是 main，当自己作为模块被调用时就是自己的名字，就相当于：我管自己叫我自己，但是在朋友眼里我就是小仙女一样。

sql 内联注释

  • /**/可以当空格用，当空格被 waf 掉时
  • /**/是 MySQL 为了兼容其他数据库搞的，当/*!xxxx*/中间有个感叹号时，xxxx 内容会被执行，xxxx 内容本意是 MySQL 自己的一些特有语句，如果是其他数据库则不会执行，MySQL 遇到了就会执行

  • /*!*/就可以用来绕 waf ，中间插入 sql 关键字绕过黑名单等等操作
  • /**/还可以用来干扰黑名单检测

这条没有自己验证过，也没找到相关资料讲这个干扰的原理的，估计是检测瓶颈，看到别人的博客有写
